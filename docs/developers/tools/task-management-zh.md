# 任务管理工具对比

Qwen Code 提供两个主要的任务管理工具：**TodoWrite** 和 **Task**。虽然它们都用于任务管理，但用途和使用场景完全不同。

## 1. TodoWrite 工具 (`todo_write`)

用于创建和管理结构化的任务列表，帮助主代理跟踪进度和组织复杂任务。

### 核心功能

- **任务状态管理**：
  - `pending`（待处理）
  - `in_progress`（进行中）
  - `completed`（已完成）

- **数据存储**：
  - 位置：`~/.qwen/todos/`
  - 按会话ID分类存储，每个会话独立的任务列表

- **实时更新**：
  - 自动更新任务状态
  - 用户可以实时看到工作进度

### 使用场景

AI 助手会在以下情况自动使用 `todo_write`：

- ✅ 需要3个或更多步骤的复杂任务
- ✅ 需要仔细规划或多个操作的复杂任务
- ✅ 用户明确要求创建待办清单
- ✅ 用户提供多个任务列表（编号或逗号分隔）
- ✅ 接收新指令后立即捕获需求为待办事项
- ✅ 开始任务前标记为 `in_progress`（同一时间只有一个任务为进行中）
- ✅ 完成任务后立即标记为 `completed` 并添加后续发现的任务

**不使用的情况**：

- ❌ 单个简单直接的任务
- ❌ 跟踪没有组织价值的琐碎任务
- ❌ 可以在少于3个简单步骤内完成的任务
- ❌ 纯对话或信息性任务

### 示例

```
用户: 我想在应用设置中添加暗黑模式切换。完成之后记得运行测试和构建！

AI 助手: 我会帮你在应用设置中添加暗黑模式切换。让我创建一个待办清单来跟踪这个实现。
*创建包含以下项目的待办清单：*
1. 在设置页面创建暗黑模式切换组件
2. 添加暗黑模式状态管理（context/store）
3. 为暗黑主题实现 CSS-in-JS 样式
4. 更新现有组件以支持主题切换
5. 运行测试和构建过程，解决任何失败或错误
*开始处理第一个任务*
```

## 2. Task 工具 (`task`)

用于将复杂任务**委托给专门的子代理（Subagents）**执行。

### 核心功能

- **动态加载子代理**：
  - 从配置中动态加载可用的子代理
  - 每个子代理有独立的工具集和配置

- **实时进度追踪**：
  - 子代理执行状态
  - 工具调用详情
  - 工具调用结果和错误
  - 整体任务进度

- **事件监听**：
  - `START` - 子代理启动
  - `TOOL_CALL` - 工具调用
  - `TOOL_RESULT` - 工具调用结果
  - `FINISH` - 子代理完成
  - `ERROR` - 子代理错误

### 使用场景

使用 `task` 工具时：

1. ✅ 需要专门AI代理处理的任务
2. ✅ 需要长时间运行的复杂任务
3. ✅ 需要后台异步执行的任务
4. ✅ 多个独立任务需要并行执行
5. ✅ 需要特定领域专业知识的任务

### 子代理特性

每个子代理可以配置：

- 特定的工具访问权限
- 专门的系统提示和指令
- 自定义模型配置
- 领域特定的知识和能力

### 示例

```
# 并行启动代码审查和测试执行
task(
  description="代码审查",
  prompt="审查用户管理模块的最近更改，检查代码质量、安全问题和最佳实践合规性。",
  subagent_type="code-reviewer"
)

task(
  description="运行测试",
  prompt="执行完整测试套件并分析任何失败。提供测试覆盖率摘要和改进建议。",
  subagent_type="test-runner"
)
```

## 主要区别对比

| 特性         | TodoWrite                                 | Task                     |
| ------------ | ----------------------------------------- | ------------------------ |
| **用途**     | 主代理跟踪自己的任务列表                  | 将任务委托给子代理执行   |
| **执行者**   | 主代理本身                                | 专门的子代理             |
| **独立性**   | 依赖主代理                                | 独立执行，有自己的工具集 |
| **状态管理** | 三种状态（pending/in_progress/completed） | 完成或失败               |
| **并发能力** | 一次一个任务进行中                        | 支持多个子代理并行执行   |
| **持久化**   | 保存到 `~/.qwen/todos/`                   | 无状态，单次执行         |
| **通信**     | 主代理与用户直接沟通                      | 只返回最终结果           |
| **工具访问** | 使用主代理的所有工具                      | 只能使用子代理配置的工具 |

## 使用建议

### 何时使用 TodoWrite

- 需要分解复杂任务为可管理的步骤
- 希望看到工作进度的实时跟踪
- 任务需要按顺序逐步完成
- 需要管理多个相关任务的状态

### 何时使用 Task

- 需要专门的领域专业知识
- 有多个独立任务可以并行执行
- 需要长时间运行的任务，不阻塞主对话
- 需要限制工具访问权限的场景
- 需要隔离工作上下文

### 组合使用

这两个工具可以组合使用：

```
# 主代理创建待办清单
todo_write(todos=[
  {content: "审查认证模块代码", status: "in_progress"},
  {content: "重构用户管理", status: "pending"},
  {content: "运行完整测试套件", status: "pending"}
])

# 使用 Task 工具委托给专门的子代理
task(
  description="代码审查",
  prompt="审查 src/auth/ 目录下的认证模块代码...",
  subagent_type="code-reviewer"
)

# 完成后更新待办清单
todo_write(todos=[...])  # 第一个任务标记为 completed
```
